## Tech Debt Summary

### Phase 3: Backend API
- **in_library=False unreachable** — All tracks are user tracks because the pipeline seeds both tables identically. The `EXISTS` subquery in search_service.py is correct code that would work if external tracks were added. Acceptable for v1 (single-user library export = everything is "in library").

### Phase 1: Infrastructure
- **validate_audio_features.py not automated** — Not called by run_pipeline.py; the flag file must exist before enrich_spotify.py runs. enrich_spotify.py defaults gracefully if missing. Low impact since audio features are currently unavailable anyway.

### Phase 4: Map View
- **Dead export: buildMapboxColorExpression** — Exported from colors.ts, never imported. MapView pre-computes colors into GeoJSON properties instead. Harmless dead code.
- **Audio features null** — Spotify endpoint restricted. Frontend shows graceful fallback. External dependency, not a code issue.

### Phase 5: Stats and Search
- **StatsSidebar country click doesn't fly map** — Opens CountryPanel but doesn't navigate the map. The fix is straightforward: use a handler like `handleSearchSelect` that also sets `flyToTarget` by looking up coordinates from the `countries` prop. Low-effort fix.

**Total: 5 items across 4 phases** — none are critical blockers.
