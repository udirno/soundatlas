<objective>
Resolve origin countries for all artists via MusicBrainz API with built-in rate limiting, disambiguation handling, and checkpoint/resume via the mb_resolution_status column.

Purpose: Country resolution is the core data needed for the world map visualization. Each artist gets mapped to their origin country (where they are FROM, not current residence). The checkpoint/resume mechanism ensures the ~50 minute pipeline run can survive interruptions.

Output: `pipeline/enrich_musicbrainz.py` (MusicBrainz country resolver)
</objective>

  <action>
Create `pipeline/enrich_musicbrainz.py` that:

1. **Setup:**
   - `musicbrainzngs.set_useragent("SoundAtlas", "1.0", "https://github.com/udirno/soundatlas")` — MUST be called before any API request.
   - Built-in rate limiting at 1 req/sec is the default in musicbrainzngs — do NOT add manual `time.sleep(1)`.
   - Connect to PostgreSQL using the `build_sync_db_url()` pattern from `seed_countries.py`. Import or copy the function.

2. **Pre-load country lookup:**
   - `SELECT iso_alpha2, id FROM countries` into a dict for fast lookup (avoid N+1 queries).

3. **Fetch pending artists:**
   - `SELECT id, name FROM artists WHERE mb_resolution_status = 'pending' ORDER BY id`
   - Log count: "Processing N pending artists".

4. **Main loop — process each pending artist:**
   - Call `musicbrainzngs.search_artists(artist=artist_name, limit=5)`.
   - Extract `artist-list` from result.
   - If empty: mark `not_found`.
   - Take top result. Check `ext:score` (integer 0-100).
   - If score < 80: mark `not_found`.
   - Normalize names for comparison (unicodedata NFKD, encode ascii ignore, decode, lowercase, strip). Compare normalized top result name against normalized search query.
   - If names don't match AND score < 95: mark `not_found`.
   - Extract `country` field (ISO alpha-2 code or None).
   - If country code exists AND is in the country lookup dict: mark `resolved`, set `country_id` and `mb_id`.
   - If country code is None or not in lookup: mark `not_found`, set `mb_id` if available.
   - **UPDATE immediately and commit** (checkpoint pattern):
     ```sql
     UPDATE artists
     SET mb_resolution_status = %s, mb_id = %s, country_id = %s, updated_at = NOW()
     WHERE id = %s AND mb_resolution_status = 'pending'
     ```
   - The `AND mb_resolution_status = 'pending'` in WHERE prevents overwriting on re-run.
   - Commit after EACH artist row (this is the checkpoint mechanism).

5. **Error handling:**
   - Catch `musicbrainzngs.WebServiceError` — log the error, mark artist as `not_found`, continue to next artist. Do NOT let one failed lookup crash the entire run.
   - Catch `musicbrainzngs.NetworkError` — log, wait 5 seconds, retry once. If still fails, mark `not_found`.

6. **Progress logging:**
   - Log every 100 artists: "Processed N/M artists (R resolved, F not_found)".
   - At completion, print summary: "MusicBrainz resolution complete: R resolved, F not_found, S skipped out of T total".

7. **CLI entrypoint** with `--env-file` arg (same pattern as seed_countries.py).

**Update `pipeline/requirements.txt`:** Add `musicbrainzngs==0.7.1` to the existing requirements.

IMPORTANT constraints per research:
- MusicBrainz `country` field is ISO alpha-2, matching `countries.iso_alpha2` directly — no conversion needed.
- The `mb_resolution_status` column IS the checkpoint — query WHERE pending, commit each row. If script crashes and restarts, it only processes remaining pending artists.
- Do NOT try to derive country from MusicBrainz `area` field — areas can be cities/regions, not countries.
- Score threshold 80 is conservative. Names like "Prince" or "The Police" may still produce wrong matches — this is acceptable for v1 (user can audit later).
