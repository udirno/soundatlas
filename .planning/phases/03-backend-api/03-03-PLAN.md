**`fuzzy_search(db: AsyncSession, q: str, limit: int = 20) -> dict`:**

Handle edge cases first:
- If `q` is None, empty, or whitespace-only, return `{"query": q or "", "artists": [], "tracks": []}`
- Strip whitespace from `q`

**Artist search query:**
```python
SIMILARITY_THRESHOLD = 0.15  # Lower than default 0.3 — music names are short/unusual

artist_stmt = (
    select(
        Artist.id,
        Artist.name,
        Artist.spotify_id,
        Artist.genres,
        Artist.image_url,
        func.similarity(Artist.name, q).label("score"),
    )
    .where(func.similarity(Artist.name, q) > SIMILARITY_THRESHOLD)
    .order_by(func.similarity(Artist.name, q).desc())
    .limit(limit)
)
```

**Track search query with in_library signal:**
Use a correlated EXISTS subquery against `user_tracks` to determine if a track is in the user's library:
```python
in_library_subq = (
    select(literal(True))
    .where(UserTrack.track_id == Track.id)
    .exists()
    .correlate(Track)
)

track_stmt = (
    select(
        Track.id,
        Track.name,
        Track.spotify_id,
        Track.album_name,
        func.similarity(Track.name, q).label("score"),
        in_library_subq.label("in_library"),
    )
    .where(func.similarity(Track.name, q) > SIMILARITY_THRESHOLD)
    .order_by(func.similarity(Track.name, q).desc())
    .limit(limit)
)
```

Execute both queries, convert to dicts via `.mappings().all()`, and return:
```python
return {
    "query": q,
    "artists": [dict(r) for r in artist_rows],
    "tracks": [dict(r) for r in track_rows],
}
```

Import `select, func` from `sqlalchemy`, `literal` from `sqlalchemy`, and models from `app.models`. Use `SIMILARITY_THRESHOLD` as a module-level constant (not mutable).

Note on `func.similarity()`: This calls PostgreSQL's `similarity()` function from the pg_trgm extension. The GIN trigram indexes on `artists.name` and `tracks.name` (created in Phase 1 migration) optimize this. No need for `SET pg_trgm.similarity_threshold` — we filter explicitly with `> SIMILARITY_THRESHOLD`.

**Update `backend/app/api/routes/search.py`** — replace the stub with the search endpoint.

Endpoint:
`GET /` → `response_model=SearchResult` — accepts `q: str = Query(...)` as required query parameter and optional `limit: int = Query(20, ge=1, le=100)`.

```python
from fastapi import APIRouter, Depends, Query
from sqlalchemy.ext.asyncio import AsyncSession
from app.database import get_db
from app.schemas.search import SearchResult
from app.services import search_service

router = APIRouter(prefix="/api/search", tags=["search"])

@router.get("/", response_model=SearchResult)
async def search(
    q: str = Query("", description="Search query string"),
    limit: int = Query(20, ge=1, le=100),
    db: AsyncSession = Depends(get_db),
):
    result = await search_service.fuzzy_search(db, q, limit)
    return result
```

Make `q` default to empty string (not required) — an empty query returns empty results, not a 422 validation error. This is friendlier for the frontend autocomplete which sends partial queries.
