  <action>
Create `backend/app/services/country_service.py` with flat async functions (NOT a class). Each function takes `AsyncSession` as first param.

**`get_country_list(db) -> list[dict]`:**
- SELECT countries with `func.count(Artist.id.distinct()).label("artist_count")` and `func.count(Track.id.distinct()).label("track_count")` via outerjoin Artist, outerjoin Track
- GROUP BY Country.id (include all Country columns in group_by)
- ORDER BY artist_count DESC
- Returns list of dicts from `.mappings().all()`
- THEN compute `top_genre` per country: run a SECOND query that loads countries with `selectinload(Country.artists)` and compute top genre via `collections.Counter` on flattened `artist.genres or []` arrays. Merge top_genre into the result dicts by country id.
- Guard against `artist.genres` being None: always use `artist.genres or []`

**`get_country_detail(db, country_id) -> dict | None`:**
- SELECT Country WHERE id == country_id with `selectinload(Country.artists).selectinload(Artist.tracks)`
- If None, return None (route raises 404)
- Compute `genre_breakdown` via `collections.Counter` on all artist genres (flattened), return top 10 as dict
- Compute `audio_feature_averages` for energy, danceability, valence, tempo, acousticness: collect values from all tracks across all artists, compute mean for non-None values, return None if all values are None for a feature
- Return dict with: id, name, iso_alpha2, latitude, longitude, artists (ORM objects — Pydantic handles serialization), genre_breakdown, audio_feature_averages

**`get_country_comparison(db, country_id) -> dict | None`:**
- Get country audio averages by reusing the detail logic (or a focused query): SELECT avg(energy), avg(danceability), avg(valence), avg(tempo), avg(acousticness) FROM tracks JOIN artists ON tracks.artist_id = artists.id WHERE artists.country_id == country_id
- Get global audio averages: SELECT avg(energy), avg(danceability), avg(valence), avg(tempo), avg(acousticness) FROM tracks
- Return dict with: id, name, iso_alpha2, country_averages, global_averages
- Both averages dicts have Optional[float] values (None if all tracks have NULL for that feature)

**`get_artist_list(db, q=None) -> list`:**
- SELECT Artist with `selectinload(Artist.country)`
- If `q` is provided, filter with `Artist.name.ilike(f"%{q}%")` (simple LIKE, not pg_trgm — the fuzzy search endpoint is separate)
- ORDER BY Artist.name
- Return list of ORM Artist objects

**`get_artist_detail(db, artist_id) -> Artist | None`:**
- SELECT Artist WHERE id == artist_id with `selectinload(Artist.tracks)` and `selectinload(Artist.country)`
- Return ORM object or None

Use `from sqlalchemy import select, func` and `from sqlalchemy.orm import selectinload`. Import models from `app.models`.

**Update `backend/app/api/routes/countries.py`** — replace the stub with full endpoints. Keep the existing `APIRouter(prefix="/api/countries", tags=["countries"])`.

Endpoints:
1. `GET /` → `response_model=list[CountryListItem]` — calls `country_service.get_country_list(db)`, returns result directly
2. `GET /{country_id}` → `response_model=CountryDetail` — calls `country_service.get_country_detail(db, country_id)`, raises `HTTPException(404)` if None
3. `GET /{country_id}/comparison` → `response_model=CountryComparison` — calls `country_service.get_country_comparison(db, country_id)`, raises `HTTPException(404)` if None

All endpoints use `db: AsyncSession = Depends(get_db)` for dependency injection. Import `get_db` from `app.database`. Import schemas from `app.schemas.country`. Import service from `app.services.country_service`.

**Update `backend/app/api/routes/artists.py`** — replace the stub with full endpoints. Keep existing router.

Endpoints:
1. `GET /` → `response_model=list[ArtistListItem]` — accepts optional `q: str = None` query param, calls `country_service.get_artist_list(db, q)` (artist functions live in country_service for simplicity since they share the same data domain)
2. `GET /{artist_id}` → `response_model=ArtistDetail` — calls `country_service.get_artist_detail(db, artist_id)`, raises `HTTPException(404)` if None

Routes are THIN — no business logic, just call service and return. All error handling (404) happens in the route, not the service.
