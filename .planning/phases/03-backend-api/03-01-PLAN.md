  <action>
Create 5 stub route files in `backend/app/api/routes/`. Each file defines an `APIRouter` with the correct prefix and tags but NO endpoint implementations yet (those come in Plans 02-04). The stubs ensure main.py can import and register all routers without import errors.

**Stub pattern for each route file:**
```python
from fastapi import APIRouter
router = APIRouter(prefix="/api/{domain}", tags=["{domain}"])
```

Prefixes:
- countries.py: `/api/countries`
- artists.py: `/api/artists`
- search.py: `/api/search`
- analytics.py: `/api/analytics`
- ai.py: `/api/ai`

**Update main.py** — add after the CORS middleware block, BEFORE the existing root/health routes:
```python
from app.api.routes import countries, artists, search, analytics, ai

app.include_router(countries.router)
app.include_router(artists.router)
app.include_router(search.router)
app.include_router(analytics.router)
app.include_router(ai.router)
```

Do NOT add a `prefix` argument to `include_router()` — the prefix is already on each `APIRouter`. Adding it would double the prefix (Pitfall 7 from research).

Keep the existing `@app.get("/")` and `@app.get("/health")` routes intact.
  </action>
