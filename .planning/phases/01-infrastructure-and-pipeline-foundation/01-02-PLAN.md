  <action>
    Create SQLAlchemy ORM models using the new DeclarativeBase pattern (from database.py Base). All models use `Mapped` and `mapped_column` type annotations.

    **backend/app/models/country.py** — Country model:
    - `id`: Integer, primary key, autoincrement
    - `name`: String(100), not null
    - `iso_alpha2`: String(2), unique, not null (ISO 3166-1 alpha-2)
    - `latitude`: Float, nullable (centroid latitude)
    - `longitude`: Float, nullable (centroid longitude)
    - `created_at`: DateTime, server_default=func.now()
    - Relationship: `artists` back_populates to Artist

    **backend/app/models/artist.py** — Artist model:
    - `id`: Integer, primary key, autoincrement
    - `name`: String(255), not null
    - `spotify_id`: String(50), unique, nullable
    - `country_id`: Integer, ForeignKey("countries.id"), nullable
    - `genres`: ARRAY(Text), nullable (PostgreSQL text[] per user decision — no join table)
    - `popularity`: Integer, nullable
    - `image_url`: String(500), nullable
    - `mb_resolution_status`: String(20), not null, server_default="pending" (per Phase 2 decision: pending/resolved/not_found/skipped)
    - `mb_id`: String(50), nullable (MusicBrainz artist MBID)
    - `created_at`: DateTime, server_default=func.now()
    - `updated_at`: DateTime, server_default=func.now(), onupdate=func.now()
    - Relationships: `country` (Country), `tracks` (back_populates)

    **backend/app/models/track.py** — Track model:
    - `id`: Integer, primary key, autoincrement
    - `name`: String(500), not null
    - `spotify_id`: String(50), unique, not null
    - `artist_id`: Integer, ForeignKey("artists.id"), nullable
    - `album_name`: String(500), nullable
    - Audio feature columns — ALL nullable Float per user decision:
      - `energy`, `danceability`, `valence`, `tempo`, `acousticness`, `instrumentalness`, `speechiness`, `liveness`
    - `duration_ms`: Integer, nullable
    - `created_at`: DateTime, server_default=func.now()
    - Relationships: `artist` (Artist)

    **backend/app/models/user_track.py** — UserTrack model (minimal per user decision):
    - `id`: Integer, primary key, autoincrement
    - `track_id`: Integer, ForeignKey("tracks.id"), not null
    - `artist_id`: Integer, ForeignKey("artists.id"), nullable
    - `added_at`: DateTime, nullable (from Spotify export if available)
    - `created_at`: DateTime, server_default=func.now()

    **backend/app/models/ai_query_log.py** — AIQueryLog model (for Phase 6):
    - `id`: Integer, primary key, autoincrement
    - `query`: Text, not null
    - `response`: Text, nullable
    - `model_name`: String(100), nullable
    - `token_count`: Integer, nullable
    - `response_time_ms`: Integer, nullable
    - `created_at`: DateTime, server_default=func.now()

    **backend/app/models/__init__.py** — Import all models:
    ```python
    from app.models.country import Country
    from app.models.artist import Artist
    from app.models.track import Track
    from app.models.user_track import UserTrack
    from app.models.ai_query_log import AIQueryLog

    __all__ = ["Country", "Artist", "Track", "UserTrack", "AIQueryLog"]
    ```

    **Alembic setup** — Initialize with async template:
    Run `cd backend && alembic init -t async alembic` to generate the async template structure.

    Then modify `backend/alembic/env.py` to:
    1. Import `from app.database import Base` and `from app.config import settings`
    2. Import `import app.models` to register all models with Base metadata
    3. Set `target_metadata = Base.metadata`
    4. Set `sqlalchemy.url` from `settings.DATABASE_URL`

    Modify `backend/alembic.ini` to set `sqlalchemy.url` to empty (will be overridden by env.py).

    **Create the initial migration** `backend/alembic/versions/001_initial_schema.py`:
    Write this as a MANUAL migration (not autogenerate) to ensure pg_trgm is created first:
    1. `op.execute('CREATE EXTENSION IF NOT EXISTS pg_trgm')` — MUST be first operation
    2. Create `countries` table
    3. Create `artists` table with FK to countries, GIN index on genres array, trgm index on name
    4. Create `tracks` table with FK to artists, trgm index on name
    5. Create `user_tracks` table with FKs to tracks and artists
    6. Create `ai_query_log` table

    Add trgm indexes for fuzzy search:
    ```python
    op.execute("CREATE INDEX ix_artists_name_trgm ON artists USING gin (name gin_trgm_ops)")
    op.execute("CREATE INDEX ix_tracks_name_trgm ON tracks USING gin (name gin_trgm_ops)")
    ```

    IMPORTANT: The migration must run against the Dockerized PostgreSQL. The DATABASE_URL in .env uses `localhost:5432` since pipeline scripts run on the host machine (per user decision).

    **pipeline/seed_countries.py** — Country seeding script:
    - Load .env from project root using python-dotenv (construct sync DATABASE_URL from POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB env vars, or provide a SYNC_DATABASE_URL override)
    - Use psycopg2 directly (not SQLAlchemy) for simplicity in pipeline scripts
    - Connect to `postgresql://soundatlas_user:soundatlas_password@localhost:5432/soundatlas_db` (host connection to Dockerized PostgreSQL)
    - Iterate over all `pycountry.countries` entries (~249)
    - For each country: extract `name`, `alpha_2` code
    - Centroid coordinates: embed a hardcoded dict of ISO alpha-2 -> (lat, lon) for all ~193 UN-recognized countries. For the remaining pycountry entries (territories, dependencies), set lat/lon to NULL. Source the centroid data from a well-known country centroid dataset (approximate centroids are fine — they're for map marker placement, not precision).

    - CLI interface: `python pipeline/seed_countries.py` with optional `--env-file` argument (defaults to `.env` in project root)
    - Print summary: "Seeded N countries (M new, K existing)"

    The centroid dict should cover at minimum all countries that could plausibly appear in a Spotify music library. Include all 193 UN member states plus common territories (Puerto Rico, Hong Kong, etc.). Use approximate geographic center coordinates rounded to 2 decimal places.
